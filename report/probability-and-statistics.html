Section 1: Core Definitions and Axioms of Probability

1.1 Sample Space and Events

Definition 1.1 (Sample Space): The sample space, denoted $\mathcal{S}$ (or sometimes $\Omega$), is the set of all possible outcomes of a random experiment.

Definition 1.2 (Event): An event $\mathcal{A}$ is a subset of the sample space: $\mathcal{A} \subseteq \mathcal{S}$.

Definition 1.3 (Event Occurrence): We say that event $\mathcal{A}$ has occurred if and only if the realized outcome $z$ satisfies $z \in \mathcal{A}$.

Reviewer's Commentary:

  • These definitions are standard and correct.
  • The notation $\mathcal{S}$ for sample space is conventional (alternative notations include $\Omega$ or $S$).
  • Events form a $\sigma$-algebra (though this technical detail is not mentioned in the source).

1.2 Kolmogorov Axioms of Probability

Definition 1.4 (Probability Function): A probability function (or probability measure) is a function $$P: \mathcal{A} \subseteq \mathcal{S} \to [0,1]$$ that satisfies the following three axioms:

Axiom 1 (Non-negativity): For any event $\mathcal{A} \subseteq \mathcal{S}$, $$P(\mathcal{A}) \geq 0$$

Axiom 2 (Normalization): The probability of the entire sample space is 1: $$P(\mathcal{S}) = 1$$

Axiom 3 (Countable Additivity): For any countable collection of mutually exclusive (disjoint) events $\{\mathcal{A}_i\}_{i=1}^{\infty}$ where $\mathcal{A}_i \cap \mathcal{A}_j = \emptyset$ for all $i \neq j$, $$P\left(\bigcup_{i=1}^{\infty} \mathcal{A}_i\right) = \sum_{i=1}^{\infty} P(\mathcal{A}_i)$$

Reviewer's Commentary:

  • These are the Kolmogorov axioms, the foundation of modern probability theory.
  • The source uses finite additivity notation but mentions "countably many" events - the full axiom requires countable additivity.
  • Important clarification: Axiom 3 requires that events be mutually exclusive (disjoint), which means $\mathcal{A}_i \cap \mathcal{A}_j = \emptyset$ for $i \neq j$. This condition must be stated explicitly.

Theorem 1.2 (Complement Rule): For any event $\mathcal{A}$ and its complement $\mathcal{A}^c = \mathcal{S} \setminus \mathcal{A}$, $$P(\mathcal{A}) + P(\mathcal{A}^c) = 1$$

Proof:

Step 1: By definition of complement, $\mathcal{A} \cap \mathcal{A}^c = \emptyset$. An event and its complement are disjoint by definition, meaning they have no outcomes in common.

Step 2: Also by definition of complement, $\mathcal{A} \cup \mathcal{A}^c = \mathcal{S}$. An event and its complement together partition the sample space, covering all possible outcomes.

Step 3: Since $\mathcal{A}$ and $\mathcal{A}^c$ are disjoint, by Axiom 3: $$P(\mathcal{A} \cup \mathcal{A}^c) = P(\mathcal{A}) + P(\mathcal{A}^c)$$ We apply the additivity property for disjoint events.

Step 4: Substitute from Step 2: $$P(\mathcal{S}) = P(\mathcal{A}) + P(\mathcal{A}^c)$$ We replace $\mathcal{A} \cup \mathcal{A}^c$ with $\mathcal{S}$ since the union of an event and its complement equals the entire sample space.

Therefore $P(\mathcal{A}) + P(\mathcal{A}^c) = 1$.

Section 2: Random Variables

2.1 What Even IS a Random Variable?

A random variable provides a mathematical framework for mapping outcomes from a sample space to numerical values, enabling quantitative analysis.

Real Example: Tossing a Coin

Consider a coin flip experiment where the sample space consists of two outcomes: "heads" and "tails". To facilitate mathematical operations, we define a random variable $X$ that maps these outcomes to numerical values:

  • Heads 1
  • Tails 0

Now we can do math! We can ask "what's the average value?" and calculate stuff.

Another Example: Rolling a Die

When you roll a die, you get faces showing dots. We define $X$ = "the number showing" which gives us values 1, 2, 3, 4, 5, or 6. This demonstrates the concept clearly.

Definition 2.1 (Random Variable - Formal Version): A random variable is a measurable function $X: \mathcal{S} \to \mathbb{R}$ that maps outcomes from the sample space to real numbers.

Formal Definition: A random variable is a measurable function that assigns a real number to each element of the sample space.

Two Types:

  • Discrete: Countable values (coin flips, dice rolls, number of customers)
  • Continuous: Any value in a range (height, weight, temperature, time)

2.2 Example: The Classic Coin Toss

Let's work through the most famous probability example - flipping a coin.

The Setup: You have a fair coin with a 50-50 chance of heads or tails. Let's flip it and see what happens.

Step 1: Define the Sample Space $$\mathcal{S} = \{\text{Heads}, \text{Tails}\}$$

These are all possible outcomes.

Step 2: Define a Random Variable Let's say X = 1 if Heads, and X = 0 if Tails.

So:

  • P(X = 1) = 0.5 (probability of heads)
  • P(X = 0) = 0.5 (probability of tails)

Step 3: Calculate the Expected Value What value do we expect on average if we flip many times?

$$E[X] = \sum_{x} x \cdot P(X = x)$$ $$= (1)(0.5) + (0)(0.5)$$ $$= 0.5 + 0$$ $$= 0.5$$

This means if we flip the coin many times and average all the results, we'll get close to 0.5.

Step 4: Calculate the Variance How spread out are the results?

Using the formula: $\text{Var}[X] = E[X^2] - (E[X])^2$

First, find $E[X^2]$: $$E[X^2] = (1^2)(0.5) + (0^2)(0.5) = 0.5$$

Then: $$\text{Var}[X] = 0.5 - (0.5)^2 = 0.5 - 0.25 = 0.25$$

Standard deviation: $$\sigma = \sqrt{0.25} = 0.5$$

The variance tells us the results are pretty spread out since you either get 0 or 1, nothing in between.

Now Let's Flip Two Coins

Sample space: $\mathcal{S} = \{HH, HT, TH, TT\}$

Let Y = total number of heads. Possible values: 0, 1, or 2.

Probabilities:

  • P(Y = 0) = 1/4 (both tails: TT)
  • P(Y = 1) = 2/4 = 1/2 (one head: HT or TH)
  • P(Y = 2) = 1/4 (both heads: HH)

Expected number of heads: $$E[Y] = (0)(1/4) + (1)(1/2) + (2)(1/4)$$ $$= 0 + 0.5 + 0.5 = 1$$

With 2 coins, on average you get 1 head.

Variance: $$E[Y^2] = (0^2)(1/4) + (1^2)(1/2) + (2^2)(1/4)$$ $$= 0 + 0.5 + 1 = 1.5$$

$$\text{Var}[Y] = 1.5 - (1)^2 = 1.5 - 1 = 0.5$$

Notice the pattern: For 1 coin, E[X] = 0.5 and for 2 coins, E[Y] = 1 = 2 0.5. Expectations add up, which demonstrates the linearity property.

2.3 More Real-Life Examples

Example 1: Weather Forecasting

When the weather person says "30% chance of rain tomorrow," what does that mean?

Sample Space: $\mathcal{S} = \{\text{Rain}, \text{No Rain}\}$

Probabilities:

  • P(Rain) = 0.3
  • P(No Rain) = 0.7

Let's define X = 1 if it rains, 0 if it doesn't.

Expected value: $$E[X] = (1)(0.3) + (0)(0.7) = 0.3$$

What this tells you: If there were 100 days exactly like tomorrow, it would rain on about 30 of them. There's a 70% chance you won't need an umbrella, and 30% chance you will.

Example 2: Exam Scores

Let's say you're taking a multiple choice test where you're either gonna:

  • Ace it (90-100): happens 40% of the time when you study
  • Do okay (70-89): happens 50% of the time
  • Barely pass (60-69): happens 10% of the time

Using the midpoint of each range:

$$E[\text{Score}] = (95)(0.4) + (80)(0.5) + (65)(0.1)$$ $$= 38 + 40 + 6.5 = 84.5$$

Expected score is 84.5, which is a solid B.

But there's variance too: $$E[\text{Score}^2] = (95^2)(0.4) + (80^2)(0.5) + (65^2)(0.1)$$ $$= 3610 + 3200 + 422.5 = 7232.5$$

$$\text{Var}[\text{Score}] = 7232.5 - (84.5)^2 = 7232.5 - 7140.25 = 92.25$$

$$\sigma = \sqrt{92.25} \approx 9.6$$

What this means: Your scores vary by about 10 points either way. Even though you expect an 84.5, you might get anywhere from 75 to 95.

Example 3: Gaming Loot Boxes

A loot box in a game has:

  • Common item (worth $1): 70% chance
  • Rare item (worth $5): 25% chance
  • Legendary item (worth $50): 5% chance

Expected value of a loot box: $$E[X] = (1)(0.70) + (5)(0.25) + (50)(0.05)$$ $$= 0.70 + 1.25 + 2.50 = 4.45$$

Each box is worth $4.45 on average. If the box costs $5 to buy, you're losing money on average - about $0.55 per box.

Checking the variance: $$E[X^2] = (1^2)(0.70) + (5^2)(0.25) + (50^2)(0.05)$$ $$= 0.70 + 6.25 + 125 = 131.95$$

$$\text{Var}[X] = 131.95 - (4.45)^2 = 131.95 - 19.8 = 112.15$$

$$\sigma = \sqrt{112.15} \approx 10.59$$

The huge variance means most of the time you'll get a $1 item, but occasionally you might hit that $50 legendary. The game company is counting on that 5% chance to keep you buying boxes.

Lesson: Expected value tells you the average, but variance tells you the risk. High variance means high unpredictability.

3.2 Conditional Probability

Definition 3.2 (Conditional Probability): The conditional probability of event $B = b$ given that event $A = a$ has occurred is defined as: $$P(B = b \mid A = a) = \frac{P(A = a, B = b)}{P(A = a)}$$ provided that $P(A = a) > 0$.

Interpretation:

  • We restrict our sample space to outcomes where $A = a$ occurred
  • We renormalize probabilities so they sum to 1 over this restricted space
  • The conditional probability measures the likelihood of $B = b$ within this restricted space

Important Condition: This definition is only valid when $P(A = a) > 0$. If $P(A = a) = 0$, conditional probability is undefined (division by zero).

3.3 Verification of Conditional Probability as Valid Probability

Theorem 3.2: For fixed $A = a$ with $P(A = a) > 0$, the function $Q(B = b) = P(B = b \mid A = a)$ satisfies all probability axioms.

Proof:

Axiom 1 (Non-negativity):

Step 1: By definition: $$Q(B = b) = P(B = b \mid A = a) = \frac{P(A = a, B = b)}{P(A = a)}$$ This follows from the definition of conditional probability.

Step 2: The numerator satisfies $P(A = a, B = b) \geq 0$. By Axiom 1 (non-negativity), joint probabilities are non-negative.

Step 3: The denominator satisfies $P(A = a) > 0$. This is given as an assumption in the problem statement.

Step 4: Therefore: $$Q(B = b) = \frac{P(A = a, B = b)}{P(A = a)} \geq \frac{0}{P(A = a)} = 0$$ The ratio of a non-negative number to a positive number is non-negative.

Axiom 2 (Normalization):

Step 1: Sum over all possible values of $B$: $$\sum_{b} Q(B = b) = \sum_{b} P(B = b \mid A = a)$$ We sum the conditional probabilities over all possible values.

Step 2: Substitute definition: $$= \sum_{b} \frac{P(A = a, B = b)}{P(A = a)}$$ This follows from the definition of conditional probability.

Step 3: Factor out constant denominator: $$= \frac{1}{P(A = a)} \sum_{b} P(A = a, B = b)$$ The denominator is constant with respect to $b$ and can be factored out.

Step 4: By marginalization (proven in Section 3.4 below): $$= \frac{1}{P(A = a)} \cdot P(A = a)$$ The sum of joint probabilities over all values of $b$ equals the marginal probability.

Step 5: Simplify: $$= 1$$ The terms cancel, yielding the desired result.

Axiom 3 (Additivity): Similar verification for disjoint events (omitted for brevity).

Therefore, conditional probability defines a valid probability measure.

4.2 Bayes' Theorem

The Big Idea: How do we update our beliefs when we get new information?

The Formula (don't run away!): $$P(A \mid B) = \frac{P(B \mid A) \cdot P(A)}{P(B)}$$

In words:

  • $P(A)$ = Prior: What we believed BEFORE seeing evidence
  • $P(B \mid A)$ = Likelihood: How likely is the evidence if our belief is true?
  • $P(B)$ = Marginal: How likely is the evidence overall?
  • $P(A \mid B)$ = Posterior: What we believe AFTER seeing evidence

REAL EXAMPLE: The Lying Friend

Your friend says "I'm coming to your party!" But you know from experience:

  • When they actually come, they ALWAYS say they're coming: $P(\text{Says Yes} \mid \text{Comes}) = 1.0$
  • But they only actually show up 30% of the time: $P(\text{Comes}) = 0.3$
  • They say "yes" 80% of the time: $P(\text{Says Yes}) = 0.8$

Question: Given an affirmative response, what is the actual probability that the individual will attend the event?

Using Bayes: $$P(\text{Comes} \mid \text{Says Yes}) = \frac{P(\text{Says Yes} \mid \text{Comes}) \cdot P(\text{Comes})}{P(\text{Says Yes})}$$

$$= \frac{(1.0)(0.3)}{0.8} = \frac{0.3}{0.8} = 0.375 = 37.5\\%$$

Result: Even though they said yes, there's only a 37.5% chance they'll actually come!

Why? Because your friend says "yes" a LOT (80% of the time), but only shows up 30% of the time. So "saying yes" doesn't mean much!

Another Example: Email Spam Filter

Your email has 2 features:

  • Prior: 20% of all emails are spam: $P(\text{Spam}) = 0.2$
  • Likelihood: If it's spam, there's a 90% chance it contains "FREE MONEY!!!": $P(\text{FREE MONEY} \mid \text{Spam}) = 0.9$
  • Marginal: Overall, 25% of emails contain "FREE MONEY": $P(\text{FREE MONEY}) = 0.25$

You get an email with "FREE MONEY!!!" - is it spam?

$$P(\text{Spam} \mid \text{FREE MONEY}) = \frac{(0.9)(0.2)}{0.25} = \frac{0.18}{0.25} = 0.72 = 72\\%$$

Boom! There's a 72% chance it's spam. Your filter flags it!

Why this works:

  1. We STARTED with "20% of emails are spam" (prior)
  2. We SAW evidence: "FREE MONEY!!!"
  3. We UPDATED our belief to "72% likely spam" (posterior)

This is exactly how spam filters, medical diagnosis, and AI work!

Section 4: Bayes' Theorem

4.1 Derivation of Bayes' Theorem

Theorem 4.1 (Bayes' Theorem - Basic Form): For events $A$ and $B$ with $P(A) > 0$ and $P(B) > 0$: $$P(A \mid B) = \frac{P(B \mid A) P(A)}{P(B)}$$

Proof:

Step 1: By definition of conditional probability: $$P(A \mid B) = \frac{P(A, B)}{P(B)}$$ This is the definition of conditional probability, valid when $P(B) > 0$.

Step 2: Note that the joint probability is symmetric: $$P(A, B) = P(B, A)$$ Joint probability is commutative, meaning $P(A \cap B) = P(B \cap A)$.

Step 3: Apply definition of conditional probability to express joint probability differently: $$P(B, A) = P(B \mid A) P(A)$$ This is the definition of conditional probability, valid when $P(A) > 0$.

Step 4: Combine Steps 2 and 3: $$P(A, B) = P(B \mid A) P(A)$$ We substitute using the symmetry of joint probability.

Step 5: Substitute Step 4 into Step 1: $$P(A \mid B) = \frac{P(B \mid A) P(A)}{P(B)}$$ We perform the substitution to obtain this expression.

Therefore Bayes' theorem is established.

Interpretation:

  • $P(A)$ is the prior probability of $A$
  • $P(B \mid A)$ is the likelihood of observing $B$ given $A$
  • $P(B)$ is the marginal probability (or evidence)
  • $P(A \mid B)$ is the posterior probability of $A$ after observing $B$

Section 6: Expectation and Variance

6.1 Expectation (Expected Value)

Definition 6.1 (Expectation - Discrete Case): For a discrete random variable $X$ with probability mass function $P(X = x)$, the expectation (or expected value, or mean) is: $$E[X] = \sum_{x} x \cdot P(X = x)$$ where the sum is over all possible values of $X$.

Conditions for existence: The expectation exists if and only if $\sum_{x} |x| \cdot P(X = x) < \infty$.

Alternative Notation: $E[X] = E_{X \sim P}[X] = \mu_X = \mu$

Definition 6.2 (Expectation of Function): For a function $f: \mathbb{R} \to \mathbb{R}$ and discrete random variable $X$: $$E_{X \sim P}[f(X)] = \sum_{x} f(x) \cdot P(X = x)$$

Definition 6.3 (Expectation - Continuous Case): For a continuous random variable $X$ with probability density function $p(x)$: $$E[X] = \int_{-\infty}^{\infty} x \cdot p(x) \, dx$$

For a function $f$: $$E_{X \sim P}[f(X)] = \int_{-\infty}^{\infty} f(x) \cdot p(x) \, dx$$

Reviewer's Note: The notation switches between discrete sums and continuous integrals, but the conceptual meaning is the same: weighted average of values.

6.2 Properties of Expectation

Theorem 6.1 (Linearity of Expectation): For random variables $X$ and $Y$ and constants $a, b \in \mathbb{R}$: $$E[aX + bY] = aE[X] + bE[Y]$$

Proof (Discrete Case):

Step 1: Write the expectation: $$E[aX + bY] = \sum_{x, y} (ax + by) \cdot P(X = x, Y = y)$$ This is the definition of expectation for a function of two random variables.

Step 2: Distribute: $$= \sum_{x, y} ax \cdot P(X = x, Y = y) + \sum_{x, y} by \cdot P(X = x, Y = y)$$ We apply the distributive property of addition over summations.

Step 3: Factor out constants: $$= a \sum_{x, y} x \cdot P(X = x, Y = y) + b \sum_{x, y} y \cdot P(X = x, Y = y)$$ Constants can be factored out of summations.

Step 4: For the first term, marginalize over $y$: $$\sum_{x, y} x \cdot P(X = x, Y = y) = \sum_{x} x \sum_{y} P(X = x, Y = y) = \sum_{x} x \cdot P(X = x) = E[X]$$ We rearrange the summation order, apply marginalization, and then use the definition of expectation.

Step 5: Similarly for second term: $$\sum_{x, y} y \cdot P(X = x, Y = y) = \sum_{y} y \cdot P(Y = y) = E[Y]$$ This follows from marginalization and the definition of expectation.

Step 6: Combine: $$E[aX + bY] = aE[X] + bE[Y]$$ We substitute the results from Steps 4 and 5 into Step 3.

Important Note: Linearity holds regardless of whether $X$ and $Y$ are independent. This is a powerful property.

Interpretation: Variance is the difference between the "mean of squares" and "square of mean."

6.5 Standard Deviation

Definition 6.6 (Standard Deviation): The standard deviation is the square root of variance: $$\sigma_X = \sqrt{\text{Var}[X]}$$

Purpose: Standard deviation is expressed in the same units as the original random variable, making it more interpretable than variance.

Example: If $X$ represents height in centimeters:

  • $\text{Var}[X]$ has units of cm (squared centimeters)
  • $\sigma_X$ has units of cm (centimeters)

Note: Standard deviation is always non-negative since it's a square root of variance (which is non-negative).

9.1 Chebyshev's Inequality

Theorem 9.1 (Chebyshev's Inequality): For any random variable $X$ with mean $\mu$ and variance $\sigma^2$, and for any $k > 0$: $$P(|X - \mu| \geq k\sigma) \leq \frac{1}{k^2}$$

Interpretation:

  • The probability that $X$ deviates from its mean by at least $k$ standard deviations is at most $1/k^2$
  • For $k = 2$: At least 75% of values lie within 2 standard deviations of the mean
  • For $k = 3$: At least 88.9% of values lie within 3 standard deviations of the mean

Note: The source states this inequality but does not provide a proof. The inequality follows from Markov's inequality applied to $(X - \mu)^2$.

Reviewer's Commentary: The inequality is correctly stated. A full proof would require:

  1. Markov's inequality: For non-negative $Y$ and $a > 0$, $P(Y \geq a) \leq E[Y]/a$
  2. Apply to $Y = (X - \mu)^2$ and $a = (k\sigma)^2$

12.1 Fundamental Proof: Success and Failure in Bernoulli Trials

Context and Motivation:

In probability theory, we often encounter situations where we conduct repeated trials with binary outcomes. Understanding the relationship between success and failure probabilities forms the foundation for analyzing more complex random processes.

Situation: Consider a random experiment with exactly two possible outcomes - success and failure. This is formalized as a Bernoulli trial.

Task: We need to establish the fundamental relationship between the probability of success, the expected value, and the probability of failure.

Mathematical Setup:

Let $X_i$ be a random variable representing a single trial where:

  • $X_i = 1$ represents success with probability $p$
  • $X_i = 0$ represents failure with probability $1-p$

Action - Detailed Proof:

Step 1: Calculate the Expected Value

By definition of expectation for a discrete random variable:

$$E[X_i] = \sum_{x} x \cdot P(X_i = x)$$

For our Bernoulli random variable:

$$E[X_i] = 0 \cdot P(X_i = 0) + 1 \cdot P(X_i = 1)$$

$$E[X_i] = 0 \cdot (1-p) + 1 \cdot p = p$$

Step 2: Verify the Relationship

Since $X_i$ only takes values 0 and 1:

  • When $X_i = 0$: the trial fails
  • When $X_i = 1$: the trial succeeds

The expected value $E[X_i] = p$ directly represents the probability of success.

Step 3: Establish the Complement Relationship

The probability of failure is:

$$P(X_i = 0) = 1 - P(X_i = 1) = 1 - p$$

This follows from the axiom that probabilities of all outcomes in a sample space sum to 1.

Result:

For a Bernoulli random variable:

  • $E[X_i] = p$ (probability of success)
  • $P(\text{failure}) = 1-p$
  • $E[X_i] + P(\text{failure}) = 1$

Real-World Application (STAR Format):

Situation: A quality control manager at a semiconductor manufacturing plant needs to assess production reliability. Each chip either passes quality standards (success) or fails (failure).

Task: Determine the long-run proportion of defective chips and plan for waste management and production capacity.

Action: The manager models each chip as a Bernoulli trial where $p = 0.97$ (success rate based on historical data). Using $E[X_i] = p$, they know that on average, 97% of chips will pass. The failure rate is $1 - 0.97 = 0.03$ or 3%.

Result: Over a production run of 1,000,000 chips, the manager expects approximately 970,000 passing chips and 30,000 defective chips. This allows proper planning for:

  • Inventory management
  • Waste disposal
  • Backup production capacity
  • Cost estimation

Limitations and Assumptions:

  1. Independence Assumption: Each trial must be independent. In reality, manufacturing defects might be correlated (e.g., if a machine degrades over time).

  2. Stationarity Assumption: The probability $p$ remains constant across trials. In practice, $p$ may change due to:

    • Machine wear
    • Operator fatigue
    • Raw material quality variations
    • Environmental conditions

  3. Binary Outcome Limitation: Real-world scenarios often have more than two outcomes (e.g., minor defects vs. major defects vs. perfect).

  4. No Time Dependency: The model assumes time between trials doesn't matter, which may not hold if there are time-dependent factors.

12.2 Detailed Proof: Sum of Random Variables and Linearity of Expectation

Context and Motivation:

When dealing with multiple random variables, we often need to find the expected value of their sum. This is fundamental in portfolio theory, signal processing, and statistical estimation.

Situation: Given $n$ random variables $X_1, X_2, \ldots, X_n$, we want to find the expected value of their sum $S_n = X_1 + X_2 + \cdots + X_n$.

Task: Prove that the expectation of a sum equals the sum of expectations, regardless of whether the random variables are independent.

Mathematical Setup:

Let $S_n = \sum_{i=1}^{n} X_i$ where each $X_i$ is a random variable with expected value $E[X_i]$.

Action - Detailed Proof:

Step 1: Apply the Definition of Expectation

For discrete random variables:

$$E[S_n] = E\left[\sum_{i=1}^{n} X_i\right] = \sum_{\text{all outcomes}} \left(\sum_{i=1}^{n} x_i\right) \cdot P(X_1 = x_1, \ldots, X_n = x_n)$$

Step 2: Interchange Summation Order

By properties of summation:

$$E[S_n] = \sum_{i=1}^{n} \sum_{\text{all outcomes}} x_i \cdot P(X_1 = x_1, \ldots, X_n = x_n)$$

Step 3: Marginalize Over Other Variables

For each $x_i$, sum over all possible values of the other variables:

$$\sum_{\text{all outcomes}} x_i \cdot P(X_1 = x_1, \ldots, X_n = x_n) = \sum_{x_i} x_i \sum_{x_1, \ldots, x_{i-1}, x_{i+1}, \ldots, x_n} P(X_1 = x_1, \ldots, X_n = x_n)$$

By marginalization, this equals:

$$= \sum_{x_i} x_i \cdot P(X_i = x_i) = E[X_i]$$

Step 4: Combine Results

$$E[S_n] = \sum_{i=1}^{n} E[X_i]$$

Special Case - Identical Distribution:

If all $X_i$ have the same distribution with $E[X_i] = \mu$:

$$E[S_n] = \sum_{i=1}^{n} \mu = n\mu$$

Result:

$$E\left[\sum_{i=1}^{n} X_i\right] = \sum_{i=1}^{n} E[X_i]$$

This property holds regardless of independence and is called the linearity of expectation.

Real-World Application (STAR Format):

Situation: An investment fund manager oversees a portfolio with 10 different stocks. Each stock has uncertain daily returns, and these returns are correlated (not independent) due to market-wide factors.

Task: Estimate the expected daily return of the entire portfolio to report to investors and make strategic allocation decisions.

Action: Let $R_i$ be the daily return of stock $i$ with expected return $E[R_i]$. The portfolio allocates weight $w_i$ to each stock. The total portfolio return is:

$$R_{\text{portfolio}} = \sum_{i=1}^{10} w_i R_i$$

Using linearity of expectation:

$$E[R_{\text{portfolio}}] = \sum_{i=1}^{10} w_i E[R_i]$$

For example:

  • Stock 1: $w_1 = 0.15$, $E[R_1] = 0.08\%$
  • Stock 2: $w_2 = 0.12$, $E[R_2] = 0.12\%$
  • ... (8 more stocks)

Result: The manager can calculate the expected portfolio return by computing the weighted sum of weighted expected returns, even though stock returns are correlated. This gives $E[R_{\text{portfolio}}] = 0.095\%$ daily, or approximately 24% annually (assuming 252 trading days). This expectation holds regardless of the correlation structure between stocks.

Limitations and Assumptions:

  1. Expectation Must Exist: For the theorem to apply, each $E[X_i]$ must be finite. Some distributions (like Cauchy distribution) have undefined expectation.

  2. Does Not Apply to Variance: While expectations add linearly, variances do NOT unless variables are independent: $$\text{Var}\left(\sum X_i\right) \neq \sum \text{Var}(X_i) \text{ (in general)}$$

  3. Does Not Capture Risk: Expected values don't account for volatility or risk. Two portfolios with the same expected return can have vastly different risk profiles.

  4. Assumes Probability Model is Correct: The calculation depends on accurate estimation of $E[X_i]$, which requires correct probability models and sufficient historical data.

  5. Discrete vs. Continuous: The proof above uses discrete variables. For continuous variables, we replace sums with integrals, but the same conclusion holds.

12.3 Detailed Proof: Alternative Formula for Variance

Context and Motivation:

Variance measures the spread of a probability distribution around its mean. While the definition $\text{Var}(X) = E[(X - E[X])^2]$ is intuitive, it's often computationally inconvenient. We need a more practical formula.

Situation: Given a random variable $X$ with mean $\mu = E[X]$, we want to find an alternative expression for variance that's easier to compute.

Task: Prove that $\text{Var}(X) = E[X^2] - (E[X])^2$.

Mathematical Setup:

Let $X$ be a random variable with:

  • Mean: $\mu = E[X]$
  • Variance (by definition): $\text{Var}(X) = E[(X - \mu)^2]$

Action - Detailed Proof:

Step 1: Expand the Squared Term

Start with the definition:

$$\text{Var}(X) = E[(X - \mu)^2]$$

Expand $(X - \mu)^2$ using the binomial formula $(a - b)^2 = a^2 - 2ab + b^2$:

$$\text{Var}(X) = E[X^2 - 2X\mu + \mu^2]$$

Step 2: Apply Linearity of Expectation

Use the fact that expectation is linear:

$$\text{Var}(X) = E[X^2] - E[2X\mu] + E[\mu^2]$$

Step 3: Simplify Terms Involving Constants

Since $\mu = E[X]$ is a constant:

$$E[2X\mu] = 2\mu E[X] = 2\mu \cdot \mu = 2\mu^2$$

$$E[\mu^2] = \mu^2 $$ (expectation of a constant is the constant itself)

Step 4: Combine Results

$$\text{Var}(X) = E[X^2] - 2\mu^2 + \mu^2$$

$$\text{Var}(X) = E[X^2] - \mu^2$$

$$\text{Var}(X) = E[X^2] - (E[X])^2$$

Step 5: Verification with Algebraic Identity

Note that this is related to the algebraic identity:

$$E[(X - E[X])^2] = E[X^2 - 2X \cdot E[X] + (E[X])^2]$$

The expectation operator passes through to give us the same result.

Result:

$$\boxed{\text{Var}(X) = E[X^2] - (E[X])^2}$$

This formula is computationally advantageous because:

  1. We can compute $E[X]$ and $E[X^2]$ independently
  2. We don't need to compute $(X - \mu)$ for each value
  3. It's numerically more stable in many cases

Real-World Application (STAR Format):

Situation: A data scientist at a ride-sharing company is analyzing trip durations to improve customer wait time predictions. They have collected 1 million trip duration records.

Task: Calculate the variance of trip durations to understand prediction uncertainty and set confidence intervals for estimated arrival times.

Action: Using the two-pass algorithm (which requires calculating the mean first, then $(X - \mu)^2$) would require:

  • Pass 1: Calculate $\bar{X} = \frac{1}{n}\sum X_i$
  • Pass 2: Calculate $\frac{1}{n}\sum (X_i - \bar{X})^2$

Instead, using the alternative formula $\text{Var}(X) = E[X^2] - (E[X])^2$:

  • Pass 1: Calculate $\bar{X} = \frac{1}{n}\sum X_i$ and $\overline{X^2} = \frac{1}{n}\sum X_i^2$ simultaneously
  • Compute: $\text{Var}(X) = \overline{X^2} - \bar{X}^2$

For example, if:

  • Average trip duration: $E[X] = 18.5$ minutes
  • Average of squared durations: $E[X^2] = 425.2$ minutes

Then: $$\text{Var}(X) = 425.2 - (18.5)^2 = 425.2 - 342.25 = 82.95 \text{ minutes}^2$$

Standard deviation: $\sigma = \sqrt{82.95} \approx 9.1$ minutes

Result: The company uses this to provide customers with confidence intervals:

  • Estimated arrival: 18.5 minutes
  • 95% confidence interval: $18.5 \pm 1.96 \times 9.1 \approx 18.5 \pm 17.8$ minutes
  • Report to customer: "Your ride will arrive in 15-25 minutes"

This analysis revealed high variability, prompting investigation into factors like traffic, time of day, and route complexity.

Limitations and Assumptions:

  1. Numerical Stability Issues: When $E[X^2]$ and $(E[X])^2$ are very close in magnitude, catastrophic cancellation can occur in floating-point arithmetic. This happens when:

    • $X$ has small relative variance
    • Values of $X$ are large in magnitude

    Example: If $X \sim 10^8$ with small variation, both terms are around $10^{16}$, but their difference (variance) might be around $10^6$, losing precision.

  2. Requires Second Moment: The formula requires $E[X^2] < \infty$. Some heavy-tailed distributions (like Pareto with certain parameters) have infinite variance.

  3. Sample Variance Correction: When estimating from samples, use Bessel's correction: $$s^2 = \frac{1}{n-1}\sum(X_i - \bar{X})^2$$ not $\frac{1}{n}$, to get an unbiased estimator.

  4. Assumes Variance Exists: Not all distributions have well-defined variance (e.g., Cauchy distribution).

  5. One-Pass Algorithm Trade-off: While computationally efficient, the formula $E[X^2] - (E[X])^2$ can be less numerically stable than the two-pass algorithm when computed naively.

12.4 Detailed Proof: Variance of a Scaled Random Variable

Context and Motivation:

In many applications, we need to scale random variables by constants (e.g., currency conversion, unit changes, linear transformations). Understanding how variance transforms under scaling is crucial for risk assessment and statistical inference.

Situation: Given a random variable $X$ with variance $\text{Var}(X)$ and a constant $c$, we want to understand how the variance of $cX$ relates to $\text{Var}(X)$.

Task: Prove that $\text{Var}(cX) = c^2 \cdot \text{Var}(X)$.

Mathematical Setup:

Let:

  • $X$ be a random variable with mean $\mu = E[X]$ and variance $\sigma^2 = \text{Var}(X)$
  • $c$ be a constant (scalar)
  • $Y = cX$ be the scaled random variable

Action - Detailed Proof:

Method 1: Using the Definition

Step 1: Find the Mean of $cX$

$$E[cX] = c \cdot E[X] = c\mu$$

(by linearity of expectation)

Step 2: Apply the Variance Definition

$$\text{Var}(cX) = E[(cX - E[cX])^2]$$

$$= E[(cX - c\mu)^2]$$

Step 3: Factor Out the Constant

$$= E[c^2(X - \mu)^2]$$

Step 4: Use Linearity of Expectation

Since $c^2$ is a constant:

$$= c^2 \cdot E[(X - \mu)^2]$$

$$= c^2 \cdot \text{Var}(X)$$

Method 2: Using the Alternative Variance Formula

Step 1: Apply the Formula $\text{Var}(Y) = E[Y^2] - (E[Y])^2$

$$\text{Var}(cX) = E[(cX)^2] - (E[cX])^2$$

Step 2: Simplify Each Term

$$E[(cX)^2] = E[c^2 X^2] = c^2 E[X^2]$$

$$(E[cX])^2 = (c \cdot E[X])^2 = c^2 (E[X])^2$$

Step 3: Substitute and Factor

$$\text{Var}(cX) = c^2 E[X^2] - c^2 (E[X])^2$$

$$= c^2 [E[X^2] - (E[X])^2]$$

$$= c^2 \cdot \text{Var}(X)$$

Result:

$$\boxed{\text{Var}(cX) = c^2 \cdot \text{Var}(X)}$$

Important Observations:

  1. Squaring Effect: Variance scales with the SQUARE of the constant, not linearly.
  2. Sign Doesn't Matter: $\text{Var}(-X) = (-1)^2 \cdot \text{Var}(X) = \text{Var}(X)$
  3. Standard Deviation: Since $\text{SD}(X) = \sqrt{\text{Var}(X)}$, we have: $$\text{SD}(cX) = |c| \cdot \text{SD}(X)$$

Real-World Application (STAR Format):

Situation: A European investment firm holds a portfolio valued in US dollars but needs to report returns to European investors in Euros. The exchange rate fluctuates daily, adding currency risk to the investment returns.

Task: Understand how currency conversion affects the volatility (risk) of returns when converting from USD to EUR.

Action: Let $R$ be the daily return in USD with:

  • Mean return: $E[R] = 0.05\%$ per day
  • Standard deviation: $\text{SD}(R) = 1.2\%$ per day
  • Variance: $\text{Var}(R) = (1.2\%)^2 = 0.000144$

The exchange rate is 1 USD = 0.92 EUR. The return in EUR is:

$$R_{\text{EUR}} = 0.92 \times R$$

Using our formula:

$$\text{Var}(R_{\text{EUR}}) = (0.92)^2 \cdot \text{Var}(R) = 0.8464 \times 0.000144 = 0.000122$$

$$\text{SD}(R_{\text{EUR}}) = 0.92 \times 1.2\% = 1.104\%$$

Mean return in EUR: $$E[R_{\text{EUR}}] = 0.92 \times 0.05\% = 0.046\%$$

Result: The firm reports to European investors:

  • Expected daily return: 0.046% in EUR (lower than USD due to conversion)
  • Risk (standard deviation): 1.104% in EUR (proportionally scaled)
  • Annual volatility: $1.104\% \times \sqrt{252} \approx 17.5\%$

This helps investors understand risk in their home currency. The risk metric scales linearly with the exchange rate, while variance scales quadratically.

Limitations and Assumptions:

  1. Assumes Constant Exchange Rate: In reality, exchange rates are themselves random variables. The true analysis requires: $$R_{\text{EUR}} = R \times E(t)$$ where $E(t)$ is the stochastic exchange rate. This makes variance calculation more complex: $$\text{Var}(R_{\text{EUR}}) \neq c^2 \cdot \text{Var}(R)$$

  2. Ignores Correlation: If returns and exchange rates are correlated, additional covariance terms appear: $$\text{Var}(R \cdot E) = E[R]^2 \text{Var}(E) + E[E]^2 \text{Var}(R) + 2E[R]E[E]\text{Cov}(R,E) + \text{higher order terms}$$

  3. Linear Transformation Only: The formula applies to linear scaling. Non-linear transformations (e.g., $Y = X^2$) have different variance relationships.

  4. No Location Shift: For affine transformations $Y = a + bX$: $$\text{Var}(Y) = b^2 \cdot \text{Var}(X)$$ The constant $a$ doesn't affect variance (shifts don't change spread).

  5. Assumes Finite Variance: Requires $\text{Var}(X) < \infty$.

12.5 Detailed Proof: Variance of Bernoulli Distribution

Context and Motivation:

The Bernoulli distribution is the simplest discrete probability distribution, modeling binary outcomes. It's the building block for binomial, geometric, and negative binomial distributions. Understanding its variance is fundamental for analyzing success/failure processes.

Situation: A random variable $X$ follows a Bernoulli distribution with parameter $p$, meaning $X = 1$ with probability $p$ and $X = 0$ with probability $1-p$.

Task: Derive the variance formula $\text{Var}(X) = p(1-p)$ and understand its properties.

Mathematical Setup:

Bernoulli random variable $X \sim \text{Bernoulli}(p)$: $$P(X = 1) = p, \quad P(X = 0) = 1-p$$

where $0 \leq p \leq 1$.

Action - Detailed Proof:

Method 1: Using the Definition

Step 1: Calculate the Mean

$$E[X] = 0 \cdot P(X=0) + 1 \cdot P(X=1) = 0 \cdot (1-p) + 1 \cdot p = p$$

Step 2: Calculate $E[X^2]$

Since $X \in \{0,1\}$, we have $X^2 = X$ (key observation: $0^2 = 0$ and $1^2 = 1$).

Therefore: $$E[X^2] = E[X] = p$$

Step 3: Apply the Variance Formula

$$\text{Var}(X) = E[X^2] - (E[X])^2 = p - p^2 = p(1-p)$$

Method 2: Direct Calculation from Definition

Step 1: Apply Definition Directly

$$\text{Var}(X) = E[(X - p)^2]$$

$$= (0 - p)^2 \cdot P(X=0) + (1-p)^2 \cdot P(X=1)$$

$$= p^2 \cdot (1-p) + (1-p)^2 \cdot p$$

Step 2: Factor and Simplify

$$= p(1-p)[p + (1-p)]$$

$$= p(1-p) \cdot 1$$

$$= p(1-p)$$

Result:

$$\boxed{\text{Var}(X) = p(1-p) = pq}$$

where $q = 1-p$ is often used as shorthand.

Properties and Insights:

  1. Symmetric in $p$ and $q$: $\text{Var}(X) = p(1-p) = (1-p)p$, so variance is the same for $p$ and $1-p$.

  2. Maximum at $p = 0.5$: Taking the derivative: $$\frac{d}{dp}[p(1-p)] = 1 - 2p = 0 \implies p = 0.5$$ Maximum variance: $\text{Var}(X) = 0.5 \times 0.5 = 0.25$

  3. Minimum at boundaries: $\text{Var}(X) = 0$ when $p = 0$ or $p = 1$ (deterministic outcomes).

  4. Standard Deviation: $\text{SD}(X) = \sqrt{p(1-p)}$

Real-World Application (STAR Format):

Situation: A pharmaceutical company is conducting clinical trials for a new drug. Each patient either responds positively to treatment (success) or does not respond (failure). The success rate in similar drugs has been 65%.

Task: Design the sample size for the clinical trial to detect treatment effects with sufficient statistical power, accounting for the inherent variability in binary outcomes.

Action: Model each patient as a Bernoulli random variable with $p = 0.65$. Calculate the variance:

$$\text{Var}(X) = p(1-p) = 0.65 \times 0.35 = 0.2275$$

$$\text{SD}(X) = \sqrt{0.2275} \approx 0.477$$

For $n = 100$ patients, the average success rate $\bar{X} = \frac{1}{n}\sum_{i=1}^{n} X_i$ has:

$$E[\bar{X}] = p = 0.65$$

$$\text{Var}(\bar{X}) = \frac{\text{Var}(X)}{n} = \frac{0.2275}{100} = 0.002275$$

$$\text{SD}(\bar{X}) = \frac{0.477}{\sqrt{100}} = 0.0477 \approx 4.77\%$$

Result: The company determines:

  1. 95% Confidence Interval: The observed success rate will fall within $0.65 \pm 1.96 \times 0.0477 \approx 0.65 \pm 0.093$, or [0.557, 0.743] with 95% probability.

  2. Sample Size Calculation: To detect a 10% improvement (from 0.65 to 0.75) with 80% power at 5% significance level, use: $$n = \frac{(z_{\alpha/2} + z_{\beta})^2 [p_1(1-p_1) + p_2(1-p_2)]}{(p_1 - p_2)^2}$$ $$n = \frac{(1.96 + 0.84)^2 [0.65 \times 0.35 + 0.75 \times 0.25]}{(0.10)^2} \approx 312 \text{ patients per group}$$

  3. Risk Assessment: The company understands that higher uncertainty (variance) at $p = 0.5$ would require larger sample sizes, while outcomes near $p = 0$ or $p = 1$ are more predictable.

Limitations and Assumptions:

  1. Independence Assumption: Assumes each trial is independent. In clinical trials, this might be violated if:

    • Patients share environmental factors
    • There are contagion effects
    • Treatment administration improves with practice

  2. Constant Success Probability: Assumes $p$ is the same across all trials. In reality:

    • Patient heterogeneity (age, genetics, comorbidities)
    • Time effects (placebo effect, drug accumulation)
    • Site differences in multi-center trials

  3. Binary Outcome Oversimplification: Real outcomes are often:

    • Partial response vs. complete response
    • Gradual improvement over time
    • Multiple endpoints (efficacy + safety)

  4. Maximum Uncertainty at $p=0.5$: This has practical implications:

    • Trials for conditions with ~50% baseline success rates require larger samples
    • Very rare diseases ($p \approx 0$) or very effective treatments ($p \approx 1$) are easier to detect
    • This seems counterintuitive but reflects that middle probabilities have maximum information entropy

  5. Normal Approximation Validity: Using normal approximation for confidence intervals requires $np > 5$ and $n(1-p) > 5$. For small $n$ or extreme $p$, use exact binomial methods.

  6. Variance Formula for Sum: For $n$ i.i.d. Bernoulli trials, the sum $S_n = \sum_{i=1}^{n} X_i \sim \text{Binomial}(n,p)$ has: $$\text{Var}(S_n) = np(1-p)$$ NOT $n^2 p(1-p)$, because variance of sum equals sum of variances only for independent variables.

12.6 Law of Large Numbers: Convergence Analysis

Context and Motivation:

The Law of Large Numbers (LLN) is one of the most fundamental theorems in probability theory and statistics. It provides the mathematical justification for why empirical averages converge to theoretical expectations, underpinning all of statistical inference.

Situation: We repeatedly sample from a probability distribution and compute the sample average. Intuitively, we expect this average to get closer to the true expected value as we take more samples.

Task: Formally prove that the sample average converges to the expected value as the number of samples approaches infinity, and quantify the rate of convergence.

Mathematical Setup:

Let $X_1, X_2, \ldots, X_n$ be independent and identically distributed (i.i.d.) random variables with:

  • Mean: $\mu = E[X_i]$
  • Variance: $\sigma^2 = \text{Var}(X_i) < \infty$

Define the sample average: $$\bar{X}_n = \frac{1}{n}\sum_{i=1}^{n} X_i$$

Action - Detailed Proof:

Weak Law of Large Numbers (WLLN)

Step 1: Calculate the Expected Value of the Sample Average

By linearity of expectation:

$$E[\bar{X}_n] = E\left[\frac{1}{n}\sum_{i=1}^{n} X_i\right] = \frac{1}{n}\sum_{i=1}^{n} E[X_i] = \frac{1}{n} \cdot n\mu = \mu$$

This shows the sample average is an unbiased estimator of $\mu$.

Step 2: Calculate the Variance of the Sample Average

Since the $X_i$ are independent:

$$\text{Var}(\bar{X}_n) = \text{Var}\left(\frac{1}{n}\sum_{i=1}^{n} X_i\right) = \frac{1}{n^2}\sum_{i=1}^{n}\text{Var}(X_i)$$

(Using the fact that variance of sum of independent variables equals sum of variances)

$$= \frac{1}{n^2} \cdot n\sigma^2 = \frac{\sigma^2}{n}$$

Step 3: Apply Chebyshev's Inequality

For any $\epsilon > 0$, Chebyshev's inequality states:

$$P(|\bar{X}_n - \mu| \geq \epsilon) \leq \frac{\text{Var}(\bar{X}_n)}{\epsilon^2} = \frac{\sigma^2}{n\epsilon^2}$$

Step 4: Take the Limit

As $n \to \infty$:

$$\lim_{n \to \infty} P(|\bar{X}_n - \mu| \geq \epsilon) \leq \lim_{n \to \infty} \frac{\sigma^2}{n\epsilon^2} = 0$$

Since probability is non-negative:

$$\lim_{n \to \infty} P(|\bar{X}_n - \mu| \geq \epsilon) = 0$$

Equivalently:

$$\lim_{n \to \infty} P(|\bar{X}_n - \mu| < \epsilon) = 1$$

Result:

$$\boxed{\bar{X}_n \xrightarrow{P} \mu \quad \text{(convergence in probability)}}$$

Convergence Rate Analysis:

From Chebyshev's inequality, to achieve accuracy $\epsilon$ with probability at least $1 - \delta$, we need:

$$\frac{\sigma^2}{n\epsilon^2} \leq \delta$$

Solving for $n$:

$$n \geq \frac{\sigma^2}{\epsilon^2 \delta}$$

Key Insights:

  1. Rate: Convergence occurs at rate $O(1/\sqrt{n})$ because: $$\text{SD}(\bar{X}_n) = \frac{\sigma}{\sqrt{n}}$$

  2. Sample Size Scaling: To double accuracy (halve $\epsilon$), need 4 times as many samples.

  3. Variance Matters: Distributions with high variance require many more samples for accurate estimation.

Real-World Application (STAR Format):

Situation: An online advertising company runs A/B tests to determine which ad variant generates higher click-through rates (CTR). Each user either clicks (1) or doesn't click (0) on the ad, making it a Bernoulli trial. The company needs to decide when to stop the test and declare a winner.

Task: Determine how many users must view each ad variant to reliably estimate the true CTR within a desired accuracy level.

Action: For Variant A with true CTR $p = 0.03$ (3%):

  • Mean: $\mu = p = 0.03$
  • Variance: $\sigma^2 = p(1-p) = 0.03 \times 0.97 = 0.0291$
  • Standard deviation: $\sigma = 0.171$

Scenario 1: Estimate CTR within $\epsilon = 0.002$ (0.2 percentage points) with 95% confidence ($\delta = 0.05$).

Using Chebyshev (conservative): $$n \geq \frac{\sigma^2}{\epsilon^2 \delta} = \frac{0.0291}{(0.002)^2 \times 0.05} = \frac{0.0291}{0.0000002} = 145,500$$

Using normal approximation (more accurate for large $n$): $$n \geq \left(\frac{1.96 \times \sigma}{\epsilon}\right)^2 = \left(\frac{1.96 \times 0.171}{0.002}\right)^2 \approx 27,889$$

Scenario 2: After $n = 10,000$ views:

  • Sample CTR: $\bar{X}_{10000} = 0.0317$ (317 clicks)
  • Standard error: $\text{SE} = \frac{0.171}{\sqrt{10000}} = 0.00171$
  • 95% CI: $0.0317 \pm 1.96 \times 0.00171 = [0.0283, 0.0351]$

Result: The company:

  1. Implements Sequential Testing: Continuously monitors the sample average and stops when confidence interval width is below threshold.

  2. Practical Decision Rule: With 10,000 samples per variant, can detect CTR differences larger than 0.5 percentage points with high confidence.

  3. Cost-Benefit Analysis: Running the test for 30,000 users costs an estimated $500 in lost revenue (if one variant is indeed better), but provides reliable data for future campaigns affecting millions of users.

  4. Observed Phenomenon: Initially, with $n = 100$, the sample CTR was 0.05 (5%), far from the true 3%. As $n$ increased to 10,000, it stabilized around 0.031-0.032, demonstrating LLN in action.

Limitations and Assumptions:

  1. Independence Requirement:

    • Violation: Users sharing the same device/account, social influence effects, time-of-day confounding
    • Impact: Effective sample size is smaller than nominal sample size
    • Mitigation: Cluster randomization, time series analysis

  2. Identically Distributed Assumption:

    • Violation: User heterogeneity (mobile vs. desktop, different geographies, returning vs. new users)
    • Impact: The "true" mean $\mu$ may not be well-defined if population is mixed
    • Mitigation: Stratified analysis, mixture models

  3. Finite Variance Requirement:

    • Critical: If $\sigma^2 = \infty$, WLLN does not apply
    • Example: Pareto distribution with $\alpha \leq 2$ has infinite variance
    • Real-world: Some financial returns exhibit heavy tails where variance may not exist
    • Solution: Use robust statistics (median instead of mean)

  4. Convergence Rate:

    • $O(1/\sqrt{n})$ is slow: To get 1 more decimal place of accuracy, need 100 more samples
    • Compare to: Deterministic algorithms often have exponential convergence
    • Implication: Monte Carlo methods require massive sample sizes for high precision

  5. Chebyshev vs. CLT Bounds:

    • Chebyshev's inequality (used in proof) is very conservative
    • Central Limit Theorem provides tighter bounds for large $n$
    • Rule of thumb: Use CLT when $n > 30$ and distribution not heavily skewed

  6. Asymptotic Nature:

    • WLLN is an asymptotic result ($n \to \infty$)
    • For finite $n$, no guarantee sample average is close to $\mu$
    • Need additional tools (confidence intervals, hypothesis tests) for practical inference

  7. Does Not Imply Almost Sure Convergence:

    • Weak LLN: $P(|\bar{X}_n - \mu| > \epsilon) \to 0$ for fixed $\epsilon$
    • Strong LLN (SLLN): $P(\lim_{n \to \infty} \bar{X}_n = \mu) = 1$
    • SLLN is stronger but requires more stringent conditions

12.7 Sample Space and Event Algebra (Sigma-Algebra)

Context and Motivation:

Before we can rigorously define probability, we need a mathematical framework to describe the "space" of all possible outcomes and which collections of outcomes we can assign probabilities to. This is the role of sample spaces and sigma-algebras.

Situation: We perform a random experiment (flip a coin, roll a die, measure temperature, etc.). We need a mathematical structure to represent all possible outcomes and events.

Task: Define the sample space, event space, and understand the sigma-algebra structure that makes probability theory rigorous.

Mathematical Framework:

Definition 12.1 (Sample Space):

The sample space $\mathcal{S}$ (or $\Omega$) is the set of all possible outcomes of a random experiment.

Examples:

  1. Coin flip: $\mathcal{S} = \{\text{H}, \text{T}\}$
  2. Die roll: $\mathcal{S} = \{1,2,3,4,5,6\}$
  3. Temperature measurement: $\mathcal{S} = \mathbb{R}$ (all real numbers)
  4. Sequence of coin flips: $\mathcal{S} = \{\text{H}, \text{T}\}^{\infty}$ (infinite sequences)

Definition 12.2 (Event):

An event $E$ is a subset of the sample space, $E \subseteq \mathcal{S}$.

Examples:

  1. Die roll $> 4$: $E = \{5,6\}$
  2. Temperature above freezing: $E = \{x \in \mathbb{R} : x > 0\}$

Definition 12.3 (Event Space / $\sigma$-Algebra):

An event space (or $\sigma$-algebra) $\mathcal{F}$ is a collection of subsets of $\mathcal{S}$ satisfying:

  1. Contains sample space: $\mathcal{S} \in \mathcal{F}$

  2. Closed under complementation: If $E \in \mathcal{F}$, then $E^c = \mathcal{S} \setminus E \in \mathcal{F}$

  3. Closed under countable unions: If $E_1, E_2, E_3, \ldots \in \mathcal{F}$, then $\bigcup_{i=1}^{\infty} E_i \in \mathcal{F}$

Why Do We Need This Structure?

In measure theory (the foundation of modern probability), not all subsets of $\mathcal{S}$ can be assigned consistent probabilities. The $\sigma$-algebra defines which events are "measurable" and can have probabilities.

Consequences of $\sigma$-Algebra Properties:

From the three axioms above, we can derive:

  1. Empty set: $\emptyset \in \mathcal{F}$ (since $\emptyset = \mathcal{S}^c$)

  2. Closed under countable intersections: $$\bigcap_{i=1}^{\infty} E_i = \left(\bigcup_{i=1}^{\infty} E_i^c\right)^c \in \mathcal{F}$$

  3. Closed under finite unions and intersections: Special case of countable operations

  4. Closed under set difference: $E_1 \setminus E_2 = E_1 \cap E_2^c \in \mathcal{F}$

Action - Proof that $\emptyset \in \mathcal{F}$:

Step 1: By axiom 1, $\mathcal{S} \in \mathcal{F}$.

Step 2: By axiom 2 (complementation), $\mathcal{S}^c \in \mathcal{F}$.

Step 3: By definition of complement, $\mathcal{S}^c = \emptyset$.

Step 4: Therefore, $\emptyset \in \mathcal{F}$.

Common Examples of $\sigma$-Algebras:

  1. Trivial $\sigma$-algebra: $\mathcal{F} = \{\emptyset, \mathcal{S}\}$

    • Smallest possible
    • No non-trivial events can be distinguished

  2. Power set: $\mathcal{F} = 2^{\mathcal{S}}$ (all subsets of $\mathcal{S}$)

    • Largest possible
    • Works for discrete/countable sample spaces
    • Too large for uncountable spaces like $\mathbb{R}$ (leads to contradictions)

  3. Borel $\sigma$-algebra on $\mathbb{R}$: $\mathcal{F} = \mathcal{B}(\mathbb{R})$

    • Generated by all open intervals $(a,b)$
    • Includes all intervals, open/closed sets, countable unions, etc.
    • Standard choice for continuous random variables

Real-World Application (STAR Format):

Situation: A financial engineer is modeling stock prices for option pricing. Stock prices are continuous (take any positive real value), and we need to compute probabilities of events like "price exceeds $150" or "price falls between $100 and $120."

Task: Construct a probability model that can handle continuous outcomes and complex events (intervals, unions, complements).

Action:

  1. Sample Space: Let $\mathcal{S} = \mathbb{R}^+ = (0, \infty)$ represent all possible stock prices.

  2. Event Space: Use the Borel $\sigma$-algebra $\mathcal{B}(\mathbb{R}^+)$, which includes:

    • All intervals: $(a,b)$, $[a,b]$, $(a,b]$, $[a,b)$, $(a,\infty)$, etc.
    • Single points: $\{x\}$ (though these have probability 0 for continuous distributions)
    • Countable unions: "Price in $[100,110] \cup [120,130] \cup \ldots$"
    • Complements: "Price NOT in $[100,150]$" = $(0,100) \cup (150,\infty)$

  3. Probability Measure: Define $P$ on $\mathcal{B}(\mathbb{R}^+)$, for example using a log-normal distribution.

  4. Event Examples:

    • $E_1 = \{x : x > 150\}$ = "Stock price above $150" This is in $\mathcal{B}(\mathbb{R}^+)$
    • $E_2 = \{x : 100 \leq x \leq 120\}$ = "Price in range" This is in $\mathcal{B}(\mathbb{R}^+)$
    • $E_3 = E_1 \cup E_2$ = "Price above $150 OR in $[100,120]$" By closure, this is in $\mathcal{B}(\mathbb{R}^+)$

Result: Using the $\sigma$-algebra framework:

  • The engineer can rigorously define and compute probabilities for complex events
  • Option prices (which depend on tail probabilities) can be calculated consistently
  • Monte Carlo simulations can be validated against theoretical probabilities
  • Risk metrics like Value-at-Risk (VaR) are well-defined

For example, to compute probability of stock exceeding $150 given current price $S_0 = 100$:

$$P(S_T > 150) = P\left(\ln S_T > \ln 150\right) = 1 - \Phi\left(\frac{\ln 150 - (\ln 100 + \mu T)}{\sigma\sqrt{T}}\right)$$

where $\Phi$ is the standard normal CDF, and this computation is valid because $\{S_T > 150\} \in \mathcal{B}(\mathbb{R}^+)$.

Limitations and Assumptions:

  1. Non-Measurable Sets Exist:

    • For uncountable spaces like $\mathbb{R}$, not all subsets can be in the $\sigma$-algebra
    • Vitali set: A famous example of a non-measurable set
    • Implication: Some "events" cannot be assigned consistent probabilities
    • Practical impact: Essentially noneall events encountered in practice are measurable

  2. Choice of $\sigma$-Algebra:

    • Different choices lead to different probability spaces
    • Too small: Cannot distinguish important events
    • Too large: May not admit consistent probability measures
    • Standard practice: Use Borel $\sigma$-algebra for metric spaces

  3. Computational Complexity:

    • Even if an event is measurable, computing its probability may be intractable
    • Example: Probability that Brownian motion hits a fractal boundary
    • Need numerical methods (Monte Carlo, PDE solvers)

  4. Countable vs. Uncountable Unions:

    • $\sigma$-algebra closed under countable unions, not uncountable
    • Example: $\bigcup_{x \in [0,1]} \{x\}$ might not be in the $\sigma$-algebra (though this specific case equals $[0,1]$ which is)
    • Matters for advanced measure theory but rarely in applications

  5. Abstract Nature:

    • For discrete problems (finite sample spaces), can ignore $\sigma$-algebra subtleties
    • Use power set: $\mathcal{F} = 2^{\mathcal{S}}$
    • All subsets are events, all events have well-defined probabilities

  6. Conditional Probability and Filtrations:

    • In dynamic systems (time series, stochastic processes), need increasing sequence of $\sigma$-algebras: $\mathcal{F}_1 \subseteq \mathcal{F}_2 \subseteq \ldots$
    • Represents information accumulation over time
    • Essential for martingale theory and stochastic calculus

Section 14: Detailed Proof and Applications of Chebyshev's Inequality

14.1 Statement and Significance

Context and Motivation:

Chebyshev's inequality provides a powerful bound on the probability that a random variable deviates from its mean. Unlike specific distribution results, it applies to ANY distribution with finite variance, making it universally applicable.

Theorem (Chebyshev's Inequality):

Let $X$ be a random variable with mean $\mu = E[X]$ and variance $\sigma^2 = \text{Var}(X) < \infty$. For any $\epsilon > 0$:

$$P(|X - \mu| \geq \epsilon) \leq \frac{\sigma^2}{\epsilon^2}$$

Equivalently:

$$P(|X - \mu| < \epsilon) \geq 1 - \frac{\sigma^2}{\epsilon^2}$$

Alternative Form (in terms of standard deviations):

For $k > 0$:

$$P(|X - \mu| \geq k\sigma) \leq \frac{1}{k^2}$$

This tells us that the probability of being $k$ standard deviations away from the mean is at most $1/k^2$.

14.2 Detailed Proof

Mathematical Setup:

Let $X$ be a random variable with $E[X] = \mu$ and $\text{Var}(X) = \sigma^2 < \infty$.

Let $\epsilon > 0$ be given.

Action - Complete Proof:

Step 1: Define an Indicator Random Variable

Define: $$I = \begin{cases} 1 & \text{if } |X - \mu| \geq \epsilon \\\\ 0 & \text{otherwise} \end{cases}$$

Note that $E[I] = P(|X - \mu| \geq \epsilon)$ by definition of expectation for indicator variables.

Step 2: Establish Key Inequality

Observe that for all outcomes:

$$I \leq \frac{(X - \mu)^2}{\epsilon^2}$$

Justification:

  • If $|X - \mu| \geq \epsilon$: Then $(X - \mu)^2 \geq \epsilon^2$, so $\frac{(X - \mu)^2}{\epsilon^2} \geq 1 = I$
  • If $|X - \mu| < \epsilon$: Then $I = 0$ and $\frac{(X - \mu)^2}{\epsilon^2} \geq 0 = I$

In both cases, the inequality holds.

Step 3: Take Expectations

Since the inequality holds pointwise, it also holds in expectation:

$$E[I] \leq E\left[\frac{(X - \mu)^2}{\epsilon^2}\right]$$

Step 4: Simplify Using Linearity

$$E[I] \leq \frac{1}{\epsilon^2} E[(X - \mu)^2]$$

Step 5: Recognize the Variance

By definition, $E[(X - \mu)^2] = \text{Var}(X) = \sigma^2$:

$$E[I] \leq \frac{\sigma^2}{\epsilon^2}$$

Step 6: Substitute Definition of $I$

Recall that $E[I] = P(|X - \mu| \geq \epsilon)$:

$$P(|X - \mu| \geq \epsilon) \leq \frac{\sigma^2}{\epsilon^2}$$

Conclusion:

This completes the proof. The beauty of this proof is that it makes NO assumptions about the distribution of $X$ beyond having finite mean and variance.

Section 15: Gaussian (Normal) Distribution - Comprehensive Treatment

15.1 Definition and Fundamental Properties

Context and Motivation:

The Gaussian (or Normal) distribution is the most important continuous probability distribution in statistics and machine learning. It appears naturally through the Central Limit Theorem, making it applicable to countless real-world phenomena.

Definition:

A random variable $X$ follows a Gaussian distribution with parameters $\mu$ (mean) and $\sigma^2$ (variance), denoted $X \sim \mathcal{N}(\mu, \sigma^2)$, if it has probability density function (PDF):

$$f(x) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

for $x \in \mathbb{R}$, $\mu \in \mathbb{R}$, and $\sigma > 0$.

Key Properties:

  1. Symmetry: $f(\mu + x) = f(\mu - x)$ for all $x$ (bell curve centered at $\mu$)

  2. Unimodal: Single peak (mode) at $x = \mu$

  3. Asymptotic: $f(x) \to 0$ as $x \to \pm\infty$ (exponentially fast)

  4. Support: $x \in (-\infty, \infty)$ (all real numbers)

  5. Parameters: $\mu$ controls location, $\sigma$ controls spread

Standard Normal Distribution:

When $\mu = 0$ and $\sigma = 1$, we get the standard normal $Z \sim \mathcal{N}(0,1)$:

$$\phi(z) = \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{z^2}{2}\right)$$

Standardization:

Any Gaussian random variable can be standardized:

$$\text{If } X \sim \mathcal{N}(\mu, \sigma^2), \text{ then } Z = \frac{X - \mu}{\sigma} \sim \mathcal{N}(0,1)$$

15.2 Proof that PDF Integrates to 1

Task: Prove that the Gaussian PDF is a valid probability density function, i.e., that:

$$\int_{-\infty}^{\infty} \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right) dx = 1$$

Proof:

Step 1: Standardize by Change of Variables

Let $z = \frac{x - \mu}{\sigma}$, so $dx = \sigma dz$:

$$\int_{-\infty}^{\infty} \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right) dx = \int_{-\infty}^{\infty} \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{z^2}{2}\right) dz$$

So it suffices to show that the standard normal integrates to 1.

Step 2: Use the Gaussian Integral Trick

Let: $$I = \int_{-\infty}^{\infty} \exp\left(-\frac{z^2}{2}\right) dz$$

We need to show that $I = \sqrt{2\pi}$.

Consider: $$I^2 = \left(\int_{-\infty}^{\infty} \exp\left(-\frac{z^2}{2}\right) dz\right) \left(\int_{-\infty}^{\infty} \exp\left(-\frac{w^2}{2}\right) dw\right)$$

Step 3: Convert to Polar Coordinates

Treat as a double integral: $$I^2 = \int_{-\infty}^{\infty} \int_{-\infty}^{\infty} \exp\left(-\frac{z^2 + w^2}{2}\right) dz\,dw$$

Convert to polar coordinates with $z = r\cos\theta$, $w = r\sin\theta$, and $dz\,dw = r\,dr\,d\theta$:

$$I^2 = \int_0^{2\pi} \int_0^{\infty} \exp\left(-\frac{r^2}{2}\right) r\,dr\,d\theta$$

Step 4: Evaluate the Radial Integral

First integrate over $r$ with substitution $u = r^2/2$, so $du = r\,dr$:

$$\int_0^{\infty} \exp\left(-\frac{r^2}{2}\right) r\,dr = \int_0^{\infty} e^{-u} du = [-e^{-u}]_0^{\infty} = 1$$

Step 5: Complete the Calculation

$$I^2 = \int_0^{2\pi} 1\,d\theta = 2\pi$$

Therefore: $$I = \sqrt{2\pi}$$

Conclusion:

$$\int_{-\infty}^{\infty} \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{z^2}{2}\right) dz = \frac{I}{\sqrt{2\pi}} = \frac{\sqrt{2\pi}}{\sqrt{2\pi}} = 1$$

This proves the Gaussian PDF is properly normalized.

15.3 Proof of Mean and Variance

Task: Prove that if $X \sim \mathcal{N}(\mu, \sigma^2)$, then $E[X] = \mu$ and $\text{Var}(X) = \sigma^2$.

Proof of Mean:

Step 1: By standardization, if $X \sim \mathcal{N}(\mu, \sigma^2)$, then $X = \mu + \sigma Z$ where $Z \sim \mathcal{N}(0,1)$.

Step 2: By linearity of expectation: $$E[X] = E[\mu + \sigma Z] = \mu + \sigma E[Z]$$

Step 3: For standard normal $Z$: $$E[Z] = \int_{-\infty}^{\infty} z \cdot \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{z^2}{2}\right) dz$$

Since the integrand is an odd function (i.e., $f(-z) = -f(z)$) and we integrate over a symmetric interval: $$E[Z] = 0$$

Step 4: Therefore: $$E[X] = \mu + \sigma \cdot 0 = \mu$$

Proof of Variance:

Step 1: Again use $X = \mu + \sigma Z$ where $Z \sim \mathcal{N}(0,1)$.

Step 2: By properties of variance: $$\text{Var}(X) = \text{Var}(\mu + \sigma Z) = \sigma^2 \text{Var}(Z)$$

(constant $\mu$ doesn't affect variance, and variance scales as $c^2$)

Step 3: For standard normal, we need $\text{Var}(Z) = E[Z^2] - (E[Z])^2 = E[Z^2]$ (since $E[Z] = 0$).

Compute: $$E[Z^2] = \int_{-\infty}^{\infty} z^2 \cdot \frac{1}{\sqrt{2\pi}} \exp\left(-\frac{z^2}{2}\right) dz$$

Step 4: Integration by Parts

Let $u = z$ and $dv = z \exp(-z^2/2) dz$, so $du = dz$ and $v = -\exp(-z^2/2)$:

$$\int z^2 \exp\left(-\frac{z^2}{2}\right) dz = -z\exp\left(-\frac{z^2}{2}\right) + \int \exp\left(-\frac{z^2}{2}\right) dz$$

Evaluating from $-\infty$ to $\infty$:

  • The boundary term vanishes: $\lim_{z \to \pm\infty} z\exp(-z^2/2) = 0$
  • The remaining integral is $\int_{-\infty}^{\infty} \exp(-z^2/2) dz = \sqrt{2\pi}$

Step 5: Therefore: $$E[Z^2] = \frac{1}{\sqrt{2\pi}} \cdot \sqrt{2\pi} = 1$$

So $\text{Var}(Z) = 1$.

Step 6: Finally: $$\text{Var}(X) = \sigma^2 \cdot 1 = \sigma^2$$

Conclusion:

The parameters $\mu$ and $\sigma^2$ in the PDF correspond exactly to the mean and variance of the distribution, which establishes the appropriateness of the notation $\mathcal{N}(\mu, \sigma^2)$.

15.4 Real-World Application: Quality Control with 68-95-99.7 Rule

Situation: A pharmaceutical company manufactures tablets with target dosage of 500 mg of active ingredient. Due to manufacturing variability, actual dosages follow a normal distribution with:

  • Mean: $\mu = 500$ mg
  • Standard deviation: $\sigma = 5$ mg

Regulatory standards require:

  • At least 95% of tablets within $\pm 10$ mg of target (490-510 mg)
  • No more than 0.3% outside $\pm 15$ mg (485-515 mg)

Task: Verify whether the manufacturing process meets regulatory standards and determine quality control limits.

Action:

Analysis Using Normal Distribution:

For $X \sim \mathcal{N}(500, 5^2)$:

Requirement 1: Within $\pm 10$ mg

Need $P(490 < X < 510)$.

Standardize: $$P(490 < X < 510) = P\left(\frac{490-500}{5} < Z < \frac{510-500}{5}\right) = P(-2 < Z < 2)$$

From standard normal tables: $$P(-2 < Z < 2) = 2\Phi(2) - 1 \approx 2(0.9772) - 1 = 0.9544 \approx 95.44\%$$

This MEETS the requirement (> 95%).

Requirement 2: Outside $\pm 15$ mg

Need $P(X < 485 \text{ or } X > 515) < 0.003$.

Standardize: $$P(X < 485) + P(X > 515) = P(Z < -3) + P(Z > 3) = 2P(Z > 3)$$

From standard normal tables: $$P(Z > 3) \approx 0.00135$$

So: $$2 \times 0.00135 = 0.0027 = 0.27\%$$

This MEETS the requirement (< 0.3%).

68-95-99.7 Rule Application:

For normal distributions:

  • Approximately 68% within $\mu \pm \sigma$ (495-505 mg)
  • Approximately 95% within $\mu \pm 2\sigma$ (490-510 mg)
  • Approximately 99.7% within $\mu \pm 3\sigma$ (485-515 mg)

Result:

  1. Compliance Confirmed: The process meets all regulatory requirements

  2. Quality Control Limits:

    • Warning limits: $500 \pm 10$ mg (2$\sigma$) 95% of production
    • Action limits: $500 \pm 15$ mg (3$\sigma$) 99.7% of production
    • Tablets outside 3$\sigma$ limits should trigger process investigation

  3. Expected Defect Rate:

    • Out of 1,000,000 tablets per month:
    • About 45,600 tablets outside $\pm 10$ mg (4.56%)
    • About 2,700 tablets outside $\pm 15$ mg (0.27%)
    • Approximately 1,350 tablets below 485 mg
    • Approximately 1,350 tablets above 515 mg

  4. Process Capability:

    • $C_p = \frac{\text{USL} - \text{LSL}}{6\sigma} = \frac{515 - 485}{6 \times 5} = 1.0$
    • This indicates a capable process (typically $C_p > 1.33$ is preferred, but $C_p = 1$ is acceptable)

  5. Cost-Benefit:

    • To reduce $\sigma$ from 5 mg to 3.33 mg (achieving $C_p = 1.5$) would require:
    • Equipment upgrades: estimated $500,000
    • But would reduce defects from 2,700/month to 30/month
    • Savings from reduced waste provide economic justification for the capital investment

15.5 Limitations and Assumptions of the Gaussian Model

1. Symmetric Assumption

Limitation: Normal distribution is perfectly symmetric around the mean. Real-world data is often skewed.

Example: Income distribution is right-skewed (long tail to the right), not normal.

Impact: Using normal model for skewed data leads to:

  • Underestimating tail probabilities on one side
  • Overestimating on the other side
  • Poor risk assessment

Alternatives: Log-normal, Gamma, Weibull distributions for right-skewed data.

2. Thin Tails (Light-Tailed Distribution)

Limitation: Normal distribution has exponentially decaying tails: $P(|X - \mu| > t) \approx e^{-t^2}$. Real data often has heavier tails.

Example: Financial returns exhibit "fat tails" extreme events (crashes, bubbles) occur more frequently than normal model predicts.

Impact of Underestimating Tail Probability:

  • Black Swan Events: 2008 financial crisis was deemed "25-sigma event" under normal model (should occur once in $10^{137}$ years)
  • Risk Management Failure: Value-at-Risk (VaR) based on normal distribution underestimates extreme losses

Real Numbers:

  • Normal model: $P(Z > 5) \approx 3 \times 10^{-7}$ (1 in 3.5 million)
  • S&P 500 daily returns: 5-sigma moves occur about 1 in 14,000 days (observed reality)
  • This is 250 times more frequent than normal model predicts!

Alternatives: Student-t distribution, Cauchy distribution, or mixture models for heavy tails.

3. Continuous and Unbounded Support

Limitation: Normal distribution assigns positive probability to all real numbers, including negative values.

Problems:

  • Non-negative Quantities: Heights, weights, prices, counts cannot be negative
  • Bounded Quantities: Proportions, probabilities, percentages must be in [0,1] or [0,100]

Example: Modeling human height with $\mathcal{N}(170, 10^2)$ cm implies $P(\text{height} < 0) > 0$, which is physically impossible.

When It Matters: If $\mu$ is many standard deviations away from the boundary, probability of violation is negligible.

Rule of Thumb: Normal approximation is acceptable if $\mu > 3\sigma$ (for lower bound) or $\mu < U - 3\sigma$ (for upper bound $U$).

Alternatives:

  • Log-normal for positive values
  • Truncated normal for bounded variables
  • Beta distribution for proportions in [0,1]

4. Independence Assumption in Multivariate Applications

Limitation: Multivariate normal assumes specific correlation structure. Real-world dependencies can be more complex.

Example: In portfolio theory, assuming joint normal returns misses:

  • Correlation breakdown during crises (all assets become correlated)
  • Non-linear dependencies (copula effects)
  • Tail dependence (extreme events occur together)

Impact: Diversification benefits are overestimated, portfolio risk is underestimated.

5. Central Limit Theorem Requirements

Limitation: The justification "by CLT" assumes:

  • Sum of many i.i.d. random variables
  • Finite variance
  • Sufficient sample size

Violations:

  • Small Sample Size: With $n < 30$, CLT approximation may be poor, especially for skewed distributions
  • Heavy-Tailed Components: If individual variables have infinite variance (e.g., Cauchy), CLT does not apply
  • Dependence: Correlated observations slow down convergence to normality

Example: Average of $n=10$ uniform random variables is NOT well-approximated by normal (noticeable deviation).

Guideline: For skewed data, need $n \geq 100$ for good normal approximation. For symmetric data, $n \geq 30$ often suffices.

6. Parameter Estimation Uncertainty

Limitation: In practice, $\mu$ and $\sigma^2$ are estimated from data, introducing additional uncertainty.

Impact:

  • Confidence intervals should use $t$-distribution, not normal, for small samples
  • Prediction intervals wider than nominal
  • Model risk: what if true distribution is not normal?

Example: With $n=20$ samples, estimating $\mu$ and $\sigma$:

  • Using $Z$ distribution: 95% CI is $\bar{X} \pm 1.96 \frac{s}{\sqrt{n}}$
  • Using $t_{19}$ distribution: 95% CI is $\bar{X} \pm 2.09 \frac{s}{\sqrt{n}}$
  • The $t$ interval is 6.6% wider, properly accounting for estimation uncertainty

7. Computational and Interpretability Issues

Limitation: Normal CDF $\Phi(x)$ has no closed form. Must use tables, approximations, or numerical integration.

Practical Implications:

  • Need software or tables for exact calculations
  • Approximations (e.g., $\Phi(x) \approx 1 - \frac{1}{2}e^{-2x^2/\pi}$ for $x > 0$) introduce error
  • Not "explainable" to non-technical stakeholders

When It Matters: High-stakes decisions (medical, financial) require exact probabilities, not approximations.
